# HOMEWORK 10.2
## Цель проекта:

Проект HOMEWORK 10.1 разработан на Python для управления банковскими продуктами.

## Установка:
1. Клонируйте репозиторий:
```
git clone https://github.com/T-Sky-Zz/hw_bank_widget
```

2. Установите зависимости:
   
В разделе Settings > Project: SkyProject >  Python Interpreter 
выберите пункт _Add Interpreter_, затем _Add Local Interpreter_.
В списке доступных для использования инструментов для работы с виртуальными окружениями выберите Poetry
Обязательно заполните два поля — _Base interpreter_ и _Poetry executable_.

**_Обратите внимание_** на пункт _**Install packages from pyproject.toml**_  
 Чтобы избежать возможных ошибок запуска и настройки виртуального окружения, нужно **_убрать_** эту галочку!


## Использование разработанных функций:

### Модуль masks содержит:
- Функцию маскировки номера банковской карты **get_mask_card_number**.
   Номер карты замаскирован и отображается в формате _XXXX XX** **** XXXX_, где _X_ — это цифра номера. 
- Функцию маскировки номера банковского счета **get_mask_account**.
   Номер счета замаскирован и отображается в формате _**XXXX_, где _X_ — это цифра номера.

#### Пример работы функции get_mask_card_number:
```
7000792289606361     # входной аргумент
7000 79** **** 6361  # выход функции
```

#### Пример работы функции get_mask_account:
```
73654108430135874305  # входной аргумент
**4305  # выход функции
```

### Модуль widget содержит:
- Функцию **mask_account_card**, которая маскирует номер карты/счета, используя функции модуля masks. 
(Функция принимает один аргумент — строку, содержащую тип и номер карты или счета).
- Функцию **get_date**, принимает на вход строку с датой в формате _"2024-03-11T02:26:18.671407"_ 
  и возвращает строку с датой в формате _"ДД.ММ.ГГГГ"_ (_"11.03.2024"_).

#### Пример работы функции mask_account_card:
Пример входных данных для проверки функции:
```commandline
Maestro 1596837868705199
Счет 64686473678894779589
MasterCard 7158300734726758
Счет 35383033474447895560
Visa Classic 6831982476737658
Visa Platinum 8990922113665229
Visa Gold 5999414228426353
Счет 73654108430135874305
```
Пример для карты:
- входной аргумент:
```commandline
Visa Platinum 7000792289606361
```
- выход функции:
```commandline
Visa Platinum 7000 79** **** 6361 
```
Пример для счета:
- входной аргумент:
```commandline
Счет 73654108430135874305
```
- выход функции:
```commandline
Счет **4305
```

### Модуль processing содержит:
- Функцию **filter_by_state**, которая принимает список транзакций и значение их состояния (опционально, 
по умолчанию 'EXECUTED'), и возвращает новый список, содержащий только те транзакции, у которых состояние 
соответствует указанному значению.
- Функцию **sort_by_date**, которая принимает список словарей и необязательный параметр, задающий порядок сортировки
    (по умолчанию — убывание), и возвращает новый список, отсортированный по дате (date).

#### Пример работы функции filter_by_state:
Пример входных данных для проверки функции:
```
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}, {'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}]
```
Выход функции со статусом по умолчанию 'EXECUTED':
```
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}]
```
Выход функции, если вторым аргументов передано 'CANCELED':
```
[{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}]
```



#### Пример работы функции sort_by_date:
Пример входных данных для проверки функции:
```
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}, {'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}]
```

### Тестирование

Для тестирования проекта используется библиотека `pytest`. 
Чтобы запустить тесты с оценкой покрытия, выполните команду: `pytest --cov` 

Тесты покрывают следующие модули и функции: 
- `masks`: функции `get_mask_card_number` и `get_mask_account`. 
- `widget`: функции `mask_account_card` и `get_date`. 
- `processing`: функции `filter_by_state` и `sort_by_date`. 
 
Покрытие тестами составляет более 90% кода проекта.
